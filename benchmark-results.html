3 Engine Comparison: TEI vs vLLM
1.01x

TEI 1.8 matches vLLM for nomic (A100)

TEI 1.8 reaches 518 chunks/s vs vLLM's 512 chunks/s for nomic-v1.5 on A100 with 500-token chunks. With FlashNomicBert enabled, TEI now matches vLLM even for medium models.

7.2x

TEI 1.8 faster for minilm6 (A100)

TEI 1.8 delivers 4,646 chunks/s vs vLLM's 648 for MiniLM-L6-v2. FlashBert on TEI 1.8 is dramatically faster for small BERT-family models (up from 2.5x with TEI 1.2).

$0.35/M

jina-nano Optimization: A10G beats A100

For jina-nano (239M params), A10G is both faster and cheaper than A100: 793 vs 761 chunks/s at half the GPU cost. The small model simply doesn't saturate A100 compute.

2.7x

120-tok chunks 2.7x faster than 500-tok

On A10G: 793 chunks/s (120-tok) vs 310 chunks/s (500-tok), making 120-tok 2.6x cheaper at $0.35/M vs $0.90/M. Best config: vLLM A10G, 120-tok, batch 512. With 10x containers: 7,927 chunks/sec.

Key insight: With TEI 1.8 and FlashBert/FlashNomicBert enabled (after CUDA compat fix), TEI now dominates for small models (7x faster than vLLM for minilm) and matches vLLM even for medium models like nomic (518 vs 512). vLLM remains the better choice for larger models (677M+ params) like jina-v5.

4 Local vs Cloud Analysis

M2 MPS is competitive

Free and fast for small/medium models. M2 MPS beats Modal T4 for both MiniLM (1,358 vs 691) and nomic (226 vs 31.2 at 120-tok). With TEI 1.8, A10G is now 2.6x faster than MPS for minilm (3,510 vs 1,358).

GPU essential for 677M+ models

jina-v5-small shows 56x speedup on A100 vs MPS (296.8 vs 5.3 chunks/s). Large models exhaust M2 unified memory bandwidth.

SSD I/O never bottlenecks

Local SSD reads at 230 MB/s, 347x faster than compute needs. Disk is never the limiting factor in any configuration tested.

M2 MPS beats Modal T4

For both minilm6 and nomic models, local M2 MPS outperforms the cheapest cloud GPU (T4 at $0.60/hr). Cloud only wins at A10G ($1/hr) and above.

7 TEI CUDA Compatibility Fix

Root cause: The directory /usr/local/cuda/compat ships stale NVIDIA driver 535.x libraries that conflict with Modal's host driver at version 580.x. This causes TEI to silently fall back to CPU inference instead of using FlashBert on GPU.

Symptom: TEI reports "Using FlashBert" but actually runs on CPU with dramatically reduced throughput.

# In your Modal image build step:
rm -rf /usr/local/cuda/compat

# This removes the stale driver 535.x libs
# Modal's host driver 580.x takes over correctly
# Result: FlashBert on GPU, full speed

Result: After removing the compat directory, TEI 1.8 correctly uses FlashBert/FlashNomicBert on GPU. This produced massive speedups over TEI 1.2: 2.9x for minilm on A100 (1,613 → 4,646/s), 3.5x on A10G (1,017 → 3,510/s), 5.6x for nomic on A100 (91.8 → 518/s), and 3.3x on A10G (77.8 → 260/s).
